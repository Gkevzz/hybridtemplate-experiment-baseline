# Implementation Boundaries

Hard rules for any contributor (human or AI) working in this repo outside Lovable.

## Forbidden

- **Do not start the Visual Bible.** It has not been authored. Do not invent a visual system, design tokens, or component visual specs that claim Visual Bible authority.
- **Do not create new editorial components** outside the E2 registry. New components require an E2 amendment, which happens in Lovable memory (`mem://editorial/e2-component-library`), not in this repo.
- **Do not implement the three Planned, uncontracted components:** `TermDefinition`, `ConceptExplainer`, `MethodologyReference`. They have no contract yet.
- **Do not introduce** total scores, aggregate numerical ratings, leaderboards, "winner" columns, "best overall" badges, "topplista" framing, or any ranking-by-single-number mechanic.
- **Do not add casino data without real sources.** Use `"Verifiering tillkommer"` placeholders. Never invent licence numbers, bonus amounts, payment methods, withdrawal times, or verification dates.
- **Do not bypass `ResponsibleGamblingBlock`** before the first commercial CTA on any page that contains one.
- **Do not use casino aesthetics** (slots, dice, jackpots, neon, gold coins, celebratory motion, near-miss simulations).
- **Do not use urgency, FOMO, countdowns, or scarcity** copy or visuals — even for genuinely time-limited offers.
- **Do not edit `docs/constitution/*` to change governance.** Those files are snapshots. To change governance, edit the canonical `mem://...` documents inside Lovable.
- **Do not force-push, rebase, amend, or squash already-published commits** on branches that may be connected to external tooling such as Lovable (see `AGENTS.md`).
- **Do not edit `src/routeTree.gen.ts`** — it is regenerated by the TanStack Router Vite plugin.

## Allowed (within the constitution)

- Apply the 22 already-contracted E2.1 components in code, within their contract semantics.
- Migrate prototype surfaces toward constitutional conformance, per E3's phase rules.
- Routine maintenance: bug fixes, accessibility fixes, performance fixes, copy fixes that respect the editorial rules, SEO metadata fixes inside the existing standards.
- Extend existing components rather than introducing new ones.

## When in doubt

Stop and ask. The simplest correct solution that respects the constitution always wins.
